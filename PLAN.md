## 1. 背景与业务价值 (Background & Business Value)

### 1.1 Jobs-to-be-Done
| 用户角色 | 当前痛点 | 期望完成的工作 |
|---|---|---|
| 印章管理员 | 需线下扫描、人工抠图处理印模，流程繁琐 | 上传印模图片后一键生成合规电子印章 PNG |
| IT 集成人员 | 需对接多个工具进行格式转换 | 纯前端页面完成全部制作流程，无需后端 |
| 业务申请人 | 不了解电子印章尺寸规范，容易出错 | 系统自动按法规锁定尺寸，确保合规输出 |

### 1.2 核心业务价值
- **合规保障**：印章尺寸严格对齐 DB11/T 918—2021 标准，减少合规风险
- **效率提升**：纯前端实现，无需后端服务，开箱即用
- **质量控制**：自动去背景抠图 + 颜色替换，输出专业透明底电子印章

## 2. 功能范围与边界 (Scope & Boundaries)

### 2.1 IN SCOPE（本期实现）
- 印模图片上传（jpg/png，≤5MB）
- 前端图像处理：去背景抠图（白底消除）、颜色替换
- 印章类型下拉选择（依据 esignRules.md）
- 印章尺寸联动（非自定义类型锁定，自定义类型 30-45mm 可编辑）
- 印章颜色选择（原色/红/蓝/黑，原色与其他重复时隐藏）
- 其他参数调节：旋转角度、图片缩放、不透明度
- 印章效果预览（带"仅供展示使用参考"水印）
- 保存下载（透明底 PNG）；**下载文件名**与当前上传文件**主名**一致、扩展名为 **`.png`**（例：`印模扫描.jpg` → `印模扫描.png`）；若无法取得有效上传名（极端情况），回退为 **`seal_{印章类型}_{yyyyMMdd}.png`**
- 初始状态交互管控：未上传图片时，非上传区域全部禁用

### 2.2 OUT OF SCOPE（本期不做）
- 后端服务或 API 接口
- 用户登录/鉴权
- 印章入库/审批流程
- 数字证书绑定
- 批量处理

## 3. 用户交互流程 (User Journey & BDD 验收标准)

### 3.1 核心流程

```
[打开页面]
    │
    ▼
[初始状态] ──── 仅上传区域可操作，其余全部禁用/蒙层
    │
    ▼
[上传印模图片] ─── 格式校验（非jpg/png → 弹出错误提示）
    │            ── 大小校验（>5MB → 弹出错误提示）
    │            ── 成功 → 显示图片预览（棋盘格透明背景）
    ▼
[解锁全部控件] ── 所有设置区域变为可操作
    │
    ▼
[用户调整参数] ── 印章类型 / 颜色 / 旋转 / 缩放 / 不透明度
    │
    ▼
[点击"查看印章效果"] ── 前端执行去背景 + 颜色替换处理
    │                 ── 右侧渲染效果图（含水印）
    │                 ── 显示"保存"按钮
    ▼
[点击"保存"] ── 导出透明底 PNG（无水印）
    │
[可随时"重置"] ── 清空所有状态，回到初始页面
```

## 4. 详细功能规格 (Detailed Feature Specs)

- **布局模式**：左右两栏，左栏（上传预览区 + 印章尺寸设置），右栏（印章效果展示 + 其他设置）
- **参考截图**：001.png（初始态）、002.png（制作态）

### 4.2 左侧 - 上传与预览区

| 元素 | 规格 |
|---|---|
| 上传组件 | 点击 **或拖拽文件** 到上传区均可，接受 image/jpeg, image/png |
| 文件大小限制 | ≤ 5MB，超出提示错误 |
| 上传提示文案 | 第1条：请在白纸上盖章后，通过扫描或拍照上传印模，确保印章清晰完整且无多余边角；第2条：支持 jpg/png 格式，建议大小 5M 以内 |
| 上传后预览背景 | 棋盘格透明背景（灰白交替），凸显 PNG 透明区域边缘 |

## 5. 前端技术实现指引 (Frontend Technical Guidance)

### 5.1 技术栈
- **框架**：Vue 3 (Composition API) + TypeScript
- **UI 组件**：Element Plus（遵循 UI_DESIGN_SPEC.md）
- **图像处理**：HTML5 Canvas API（纯前端，无需后端）
- **构建工具**：Vite

### 5.2 核心图像处理算法要求

#### 去背景抠图（白底消除）
- 对上传图片的每个像素执行亮度/色相判断
- 像素亮度 > 阈值（建议 240/255）且饱和度低时，将 Alpha 通道置 0（透明）
- 支持边缘羽化处理（避免锯齿），羽化半径 1-2px

#### 颜色替换
- 在去背景后的印章像素上执行 HSL 色相替换
- 将原有色相替换为目标颜色色相，保留像素明度/饱和度结构
- 原色模式：不进行颜色替换，直接输出原始颜色

#### 原色主色识别
- 对非透明像素进行颜色直方图统计
- 取出现频率最高的色相区间代表色
- 与红/蓝/黑三色进行 ΔE 色差比较（Lab 空间）

### 5.3 Canvas 渲染流程
```
原始图片
    ↓ 根据虚线框与位移参数映射到原图坐标
    ↓ 执行真实裁剪（得到印章目标区域）
    ↓ 去背景抠图（白底消除）
    ↓ 颜色替换（按选择的印章颜色）
    ↓ 旋转变换（rotation angle）
    ↓ 应用不透明度（globalAlpha）
    ↓ 按目标物理尺寸(mm→px)输出到预览 Canvas（叠加水印）
    ↓ 保存时输出无水印 Canvas → toDataURL('image/png') → 下载
```

### 5.4 尺寸换算与输出规格
- **换算基准**：严格遵循 esignRules.md，PDF 文件分辨率 72dpi → 1cm = 28.346px → **1mm = 2.8346px**
- **常用印章换算参考**：
  - 42mm（企事业单位公章）≈ **119px**（42 × 2.8346 = 119.0532）
  - 40mm（机构/合同章）≈ **113px**（40 × 2.8346 = 113.384）
  - 30mm（财务专用章）≈ **85px**（30 × 2.8346 = 85.038）
  - 45mm×30mm（分支机构公章）≈ **127×85px**
  - 20mm（电子名章）≈ **57px**（20 × 2.8346 = 56.692）
- **输出格式**：8位透明通道 PNG（RGBA，无有损压缩），PNG 元数据分辨率标注 ≥ 300dpi
- **文件命名规则**：`seal_{印章类型}_{时间戳yyyyMMdd}.png`，例：`seal_企事业单位公章_20260409.png`

## 7. 风险与约束 (Risks & Constraints)

| 风险项 | 影响 | 缓解措施 |
|---|---|---|
| 图片背景复杂（非纯白） | 抠图效果不佳 | 上传提示明确要求"白纸上盖章"；提供阈值调节能力（可后续扩展） |
| 浏览器 Canvas CORS 限制 | 本地文件不受影响；若图片来自 URL 可能受限 | 本期仅支持本地文件上传，规避 CORS |
| 低端设备 Canvas 性能 | 大图处理慢 | 限制上传大小 ≤ 5MB；处理时显示 loading 状态 |
| 颜色识别精度 | 原色判断不准确 | 提供明确色差阈值，边界情况默认不显示原色 |
