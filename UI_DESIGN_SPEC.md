# 🎨 集中化电子签章管理系统 - UI 交互设计规范

## 1. 🎯 设计理念 (Design Philosophy)

电子签章系统作为集团级核心安全密码资产枢纽，UI 设计需传递 **“安全、严谨、高效、合规”** 的视觉感受。界面表达需克制，数据展示需清晰，高危与敏感操作必须有明确的阻断与警示体验。

## 2. 🛠 前端技术底座 (Tech Stack)

* **核心框架**：Vue 3 (Composition API) + TypeScript
* **UI 组件库**：Element Plus (按本规范深度定制 Theme)
* **状态管理**：Pinia
* **签署渲染引擎**：PDF.js / HTML5 Canvas (用于 PDF 底层渲染与印章图层防碰撞寻址)
* **可视化图表**：ECharts (用于运营统计大屏与全网风控拓扑图)

## 4. 📐 布局与尺寸体系 (Layout & Dimensions)

本系统**统一采用“纯左右侧边栏树状布局”**，摒弃顶部一级导航菜单。所有路由层级通过左侧菜单树逐级展开，确保深层级功能的导航效率与工作区视野最大化。

### 4.1 左侧统一导航栏 (Sidebar)

* **尺寸与交互**：默认展开宽度 `220px`，收起折叠宽度 `64px`。高度 `100vh` 铺满屏幕左侧。
* **背景与样式**：深色主题 `#001529`（传递安全厚重感），右侧无边框线。
* **顶部品牌区**：高度 `60px`，居中展示系统 Logo 与“电子签章管控中心”字样。折叠时仅展示 Logo。
* **菜单区域**：采用全局统一的树状折叠菜单 (`el-menu` tree 结构)。基于 RBAC 权限动态渲染菜单节点（例如：应用管理员登录时，隐藏“印章制作”节点）。

### 4.2 右侧工作区 (Main Workspace)

* **顶部工具栏 (Header Utility)**：
* 高度：`60px`，背景纯白 `#FFFFFF`，底部带浅色投影 `0 1px 4px rgba(0,21,41,0.08)`。
* 布局边界：**左侧**展示控制侧边栏折叠的汉堡图标 (`Hamburger`) 及当前页面的面包屑导航 (`Breadcrumb`)；**右侧**排列全局搜索、消息铃铛（带未读红点）、用户头像及当前角色切换下拉菜单。


* **核心内容区 (Content Area)**：
* 外边距：全局统一 `padding: 24px`。
* 背景与承载：底层背景色 `#F0F2F5`。核心业务视图使用 `<el-card>` 承载，卡片背景 `#FFFFFF`，圆角 `8px`，去除默认边框线。内容区支持独立滚动 `overflow-y: auto`。

## 5. 👆 核心业务交互规范 (Core Business Interactions)

### 5.1 一般级第4小级 - 数据脱敏与特权查阅 (DDM)

* **脱敏默认展示**：所有台账表格中的手机号、身份证号、敏感 IP 归属地，默认以掩码显示，字体颜色使用 `#BFBFBF`。
* **特权解密交互**：
* 有权限的用户（如印章管理员、支撑管理员），在脱敏字段右侧悬停可显示 `👁️` (眼睛图标)。
* 点击图标弹出二次鉴权弹窗，提示：“查看明文需进行短信二次验证，操作将记录特权审计日志”，系统向当前账号绑定的手机号下发短信验证码。
* 用户输入验证码通过后，前端 Token 获批提权（`Privilege Granted`），页面涉敏字段刷新为明文高亮展示。
* **Token 生命周期持续有效**：在 Token 有效期内（直至退出登录或 Token 过期），该用户浏览所有涉敏列表均直接展示明文，**不再倒计时恢复掩码**；退出登录或 Token 过期后自动恢复掩码。
* 后端在提权 Token 有效期内对每一次涉敏接口请求持续记录特权审计流水（与 `Module_A_Proposal.md §3.1` 行为定义保持一致）。



### 5.2 金库模式双人复核交互 (Maker-Checker)

* **高危按钮样式**：触发全局明文数据导出、注销核心印章的按钮，必须带有**金库锁图标 (`🔒`)**，且颜色使用警告色 `#FAAD14`。
* **阻断与轮询流程**：
* 点击后，前端拦截常规业务请求，弹出阻断式大弹窗 (`Modal`)。
* 弹窗提示：“此操作触碰安全红线，需局方安全员扫码复核”。
* 弹窗内展示动态二维码或 OTP 输入框。界面呈现 Loading 状态，后台通过 Websocket 或长轮询等待指令。
* 接收到复核通过信号后，弹窗变为绿色成功态，自动触发下载流或状态变更操作。



### 5.3 表格与台账交互 (Ledgers)

* **固定列机制**：签署文件台账、印章资产台账表格的首列（核心名称/文件流）和尾列（操作区）必须固定 (`fixed="left/right"`)。
* **拖拽列宽**：支持拖动表头分割线调整列宽，以适应超长的“统一社会信用代码”或“文件 Hash 值”显示。
* **溢出省略**：所有非关键文本必须添加 `show-overflow-tooltip` 属性，单行省略，悬停查看全量内容。

## 6. 🖼 专属业务组件样式规范 (Domain-Specific Components)

### 6.1 印章资产卡片 (Seal Asset Card)

用于“印章管理者工作台”的主视图展现。

* **容器形态**：宽 `280px`，圆角 `8px`，悬停时增加浮起动效 `transform: translateY(-2px)` 与加深阴影。
* **印章图案区**：高度 `160px`，背景为浅灰色 `#FAFAFA` 的透明网格底纹（用于凸显 PNG 透明图层的真实边缘）。居中展示印章，最大高宽不超过 `120px`。
* **状态角标**：卡片右上角绝对定位悬浮 `el-tag`（绿色“生效中”，红色“已注销”，灰色“已挂起”）。
* **证书倒计时**：底部展示底层 CA 证书有效期。剩余 `< 30天` 时，文字变为 Error 色 `#FF4D4F` 并闪烁警告。

### 6.2 签署证据链时间轴 (Evidence Timeline)

用于展示一份文件的签署全生命周期与哈希存证。

* **布局**：详情页采用双栏布局。左侧 `70%` 展示文件 PDF 缩略图摘要；右侧 `30%` 展示垂直固定的证据链时间轴 (`el-timeline`)。
* **定制节点节点**：
* **身份/意愿节点**：蓝色节点，记录操作人、登录 IP、活体比对相似度等前置动作。
* **落印节点**：使用印章红 `#E60000` 节点，带有微缩印章 Icon，展示印章名称和国家授时中心 TSA 时间戳。
* **存证节点**：绿色节点，展示长字符串的区块链 Hash 值，必须提供“一键复制”图标按钮。



### 6.3 智能排版与签署画板 (Sign Canvas Editor)

* **全屏沉浸模式**：点击【发起签署】进入画板时，隐藏左侧菜单和顶部导航，提供最大化的 PDF 渲染与拖拽空间。
* **资产侧边栏**：屏幕右侧抽屉式展示当前用户被授权可用的印章/签名列表，支持按住拖拽 (`Drag & Drop`) 至左侧 PDF 画布。
* **防碰撞占位反馈**：当拖拽印章进入已有签名的区域时，画布上实时出现红色的“排斥虚线框”；排版引擎自动计算就近的空白坐标，生成绿色的“吸附虚线框”引导用户松手落印。

### 6.4 右滑抽屉式表单 (Drawer Forms)

所有的向导式操作（如“新建印章申请”、“接口授权配置”）统一从屏幕右侧滑出 `el-drawer`。

* **宽度**：常规表单 `600px`；涉及多图片（营业执照、授权书）上传预览的表单 `800px`。
* **底部锁定操作栏**：包含【取消】与【提交审批】按钮的操作栏必须 `position: absolute` 固定在抽屉底部，确保长表单在 Y 轴滚动时，提交按钮始终可见。
